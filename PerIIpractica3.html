Cambios realizados en el repositorio (Métodos principales)

Usuarios

Modelo:

Controlador:

• Listado de toda la tabla

Para el controlador lo que se realiza es colorar el tipo de consulta que es con [HttpGet] y se coloca la ruta a utilizar luego se crea la clase donde se crea una variable que contendrá los datos de la consulta y los comparara en el IF si esta vacía o no. Dentro de la consulta con el FROM se coloca un nombre de una variable que contendrá el contexto “e” para poder extraer dato por datos y se coloca un “JOIN” para unir la tabla y no mostrar únicamente el ID de la llave foránea, luego para hacer el llamado a los datos de la otra tabla se realiza un “SELECT” nuevo donde se agregan los campos que serán la nueva consulta.

• Búsqueda por ID.

Para el método por ID no cambia mucho al método anterior, únicamente un “WHERE” que comprara el dato que se le esta enviando y muestra el dato si este existe.

• Insertar un nuevo registro.

Para insertar datos se especifica que los datos serán dados desde el cuerpo de la pagina y se coloca la referencia a que se enviaran los datos obtenidos (Serán insertados a los datos del modelo que serán enviados a la base de datos posteriormente), luego se crea una nueva variable que contendrá los datos de la consulta realizada en la misma variable y que a su vez comparara si no existe un usuario con el mismo nombre, si existe este dará un error y si no insertara el dato sin ningún problema.

• Modificar un registro existente.

Para el siguiente método lo que se realiza es colocar la ruta principal y darle el tipo de método, luego se especifica que se enviaran los datos del cuerpo y se llama la clase del modelo que contendrán los datos temporalmente y luego se realiza la consulta al igual que en el método de insertar y luego con el método “Modified” se insertaran los datos especificando anteriormente que campo de la comparación pertenece al de la base de datos.

Para los siguientes métodos de las otros controladores y modelos se realiza el mismo proceso sin excepción, por ello solo se muestran con imágenes el como quedaron y no con explicación.
